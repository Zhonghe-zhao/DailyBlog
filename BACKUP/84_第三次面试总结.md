# [第三次面试总结](https://github.com/Zhonghe-zhao/DailyBlog/issues/84)

这次的面试官并没有询问我关于语言层面的问题, 而是聚焦于一些基础的知识点, 考察了对于不同场景对于技术的理解和使用, 像我在简历中提到的高并发, 以及自己对于项目的理解, 在项目中遇见的问题, 

1. 操作系统基础和并发模型理解

进程和线程的区别, 在不同场景想对于进程和线程的使用, 进程和线程的 栈 和 堆 的分别

2. 线程内存模型

线程拥有自己的栈, 堆是和进程共享

共享的堆数据会造成数据的竞争, 面试官询问了我对于锁的知识

3. 你知道的锁的类型有哪些？

我只回答了 自旋锁和互斥锁, 然后面试管问我知道自旋锁吗？ 你知道自旋锁是做什么的吗？

4. 考察了不同的场景你应该使用进程锁还是线程锁？
（1） 在一个进程内部并发读取多个文件是什么锁？
（2） 在不同的节点访问数据库是使用什么锁？
（3） 在数据库内部对于同一个表的并发操作使用什么锁

考察了我某些场景是不是必须用什么锁，因为即使我们知道进程的开销很大但是我们还是需要使用进程 和 进程锁

5. 还考察了我在一些场景 问我是使用多进程 还是 多线程？

可能言外之意的意思是 设计一个并发系统 使用多进程还是多线程？（这个需要根据什么来选择？ 我只回答了 进程的隔离性和线程的轻量切换）
